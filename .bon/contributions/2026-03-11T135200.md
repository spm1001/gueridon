KillMode=control-group is the defining constraint for shutdown persistence. systemd sends SIGTERM to every process in the cgroup simultaneously — not sequentially. This means by the time the bridge's SIGTERM handler runs, CC child processes may have already exited and their exit handlers already fired, clearing session state (process=null, turnInProgress=false). Any data you need to persist about child process state must be snapshotted at the very top of the shutdown handler, before any other work, because the exit callbacks are racing you. The fix pattern: capture the state you need (activeTurnFolders) as the first line of shutdown(), then pass that snapshot to the persist function rather than reading from the (now-mutated) session objects. This same race would affect any future feature that needs to know "what was CC doing when we died."
