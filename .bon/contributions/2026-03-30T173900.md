folderName is a routing contract, not a display label. SSE events carry it as a key, and the client silently discards any event where the folder field doesn't match its local state. When scanFolders changed from flat names ("gueridon") to relative paths ("batterie/gueridon"), session creation still used basename() — producing a name the client would never accept. The fix (deriveFolderName stripping SCAN_ROOT prefix) was trivial, but the lesson is structural: any code that assigns folderName must produce exactly what scanFolders returns. The coupling is invisible in the type system because both are plain strings. An integration test that creates a nested folder, connects a session, and asserts the SSE event's folder field is now the regression gate.
